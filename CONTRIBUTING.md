# Contributing to Synth

Thank you for your interest in Synth.

## Issues

Issues are welcome and encouraged. If you've found a bug, have a feature request, or want to suggest an improvement, please open an issue.

When filing an issue, please include:

- **Bug reports**: Steps to reproduce, expected vs actual behavior, environment details (OS, Node version, Synth version)
- **Feature requests**: What problem you're trying to solve and why existing functionality doesn't address it

## Pull Requests

Synth is not currently accepting pull requests. The project is in active early development and requires tight control over architecture and direction to maintain velocity and coherence.

This may change in the future. For now, the best way to contribute is through well-described issues.
